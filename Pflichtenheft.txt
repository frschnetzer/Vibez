Projekt Praktikum
Pflichtenheft
1. Ausgangssituation
Die Applikation soll das Organisieren einer Party/ eines Events vereinfachen.

2. Zielsetzung
Eine simple Web-Applikation, in der Partys / Events organisiert werden können.

Der Erfolg des Projektes ist gegeben, wenn die nacher erwähnten Funktionen realisiert wurden.

Folgender Termin gilt als Abgabetermin: 02.02.2024

3. Produkteinsatz
Das Produkt ist für die Organisation eines Events gedacht.

4. Funktionale Anforderungen
    • Die Applikation wird in Blazor Server App realisiert
    • es wird MSSQL als Datenbank verwendet
    • die Datenbank soll min. drei Tabellen enthalten. User, Password, Event eventuell Email.
    • beim aufrufen der web-App wird eine Main Page mit Logo und eventuell Facts über die Applikation angezeigt. Außerdem gibt es einen Button für den Login oder Registrierung
    • Beim Login gibt es die Möglichkeit sich mit Email und Passwort anzumelden. Falls noch kein Konto erstellt wurde, soll es die Option geben, einen neuen User zu registrieren.
    • Um einen neuen User zu registrieren, müssen Email und Passwort (2x) angegeben werden.
    • Sobald man einen User erstellt hat und sich eingeloggt hat, soll wieder die Main Page angezeigt werden. Nun hat man die Option mittels eines Buttons ein neues Event zu erstellen. In der Main Page werden dann auch mehrere Events angezeigt.
    • Wenn auf den Button „Neues Event“ geklickt wurde, soll ein Pop up angezeigt werden, die dem User ermöglicht gewisse Einstellungen zu setzten.
Mögliche Einstellungen sind: Name des Events, Ort und Zeitpunkt, Gäste sollen hinzugefügt und entfernt werden können, und zum Schluss soll dann ein Preis pro Person und Insgesamt angezeigt werden.
    • Sobald auf „Speichern“ geklickt wurde, wird das Pop-up geschlossen und es wird wieder die Main Page mit allen gültigen Events angezeigt. Ein Event ist nur dann Gültig, wenn das Datum noch nicht abgelaufen ist. Nun bekommen die Gäste eine Einladung per Email.
    • Außerdem gibt es die Option ein Event anzuzeigen und auch zu bearbeiten oder eventuell zu löschen.
    • Wenn nun auf Event anzeigen geklickt wird, werden der Event Name, Ort und Zeitpunkt, Anzahl der Gäste und die Hochrechnung angezeigt.
    • Beim Klick auf bearbeiten, wird das gleiche Pop-up wie beim erstellen eines Events angezeigt, nur sind die Felder schon richtig ausgefüllt.
    • Beim Löschen eines Events soll der User nochmals gefragt werden, ob er wirklich löschen will.

Diese Funktionen sollen so gut wie möglich in der Projekt Struktur aufgeteilt werden.

5. Nicht funktionale Anforderungen
    • Gäste können Einladung zu oder absagen
    • Gäste können über die Einladung +1 angeben
    • der Standort wird per Google Maps angezeigt
    • Drei Stunden bevor dem Event soll nochmals an alle Gäste eine erinnerung gesendet werden

6. Lieferumfang
Es soll die Applikation so geliefert werden, dass sie auf den lokalen Rechnern der Programmiere des Teams funktioniert.
Im Lieferumfang ist nicht unbedingt enthalten, dass die Applikation öffentlich benutzbar ist.

7. Offene Punkte, die noch zu klären sind
    • Aufgabenaufteilung
    • Design der Applikation
    • Meilensteine
    • Datenbankdesign
    • Projektstruktur

Wichtige Entscheidungen werden im Team zusammen beschlossen. Kleinere Tasks werden selber beschlossen oder auch mit Absprache des jeweiligen anderen Teammitgliedes.

Veränderungen im Lastenheft sind so gut wie möglich zu vermeiden. Falls doch welche sind, sollen diese im Team besprochen werden und gemeinsam eingetragen werden.

8. Abnahmekriterien und Qualitätsanforderungen
Sobald die gesetzten Anforderungen erfüllt worden sind, wird die Projektleitung entlastet mit Abstimmung des Projektteams.
Kriterien sind:
    • ist es möglich einen neuen User zu registrieren
    • ist es möglich sich mit einem User anzumelden
    • können neue Events erstellt werden
    • bekommen die Gäste die Einladungen
    • werden die Daten von User und Events in einer Datenbank korrekt gespeichert
